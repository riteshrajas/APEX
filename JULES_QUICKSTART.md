# Google Jules Integration - Complete Guide

## 🎯 What You've Got

You now have **3 scripts** that enable Google Jules AI to analyze your APEX codebase and generate features:

### 1. **Setup-Jules.ps1** 🔧
Interactive setup wizard that guides you through:
- Creating a Jules account
- Connecting your APEX repository
- Generating an API key
- Saving it securely
- Testing the connection

**Run once:** `.\Setup-Jules.ps1`

### 2. **Analyze-Features-Jules.ps1** 🚀
Main analysis script that:
- Uses your Jules API key
- Creates a Jules session
- Sends APEX codebase for AI analysis
- Waits for Jules to complete
- Saves results to `conductor/feature_opportunities.md`
- Creates GitHub issue with findings
- Commits and pushes to GitHub

**Run anytime:** `.\Analyze-Features-Jules.ps1`

### 3. **JULES_GUIDE.md** 📖
Complete documentation covering:
- Setup instructions
- Usage examples
- Troubleshooting
- Advanced options
- CI/CD integration
- FAQ

## 🚀 Quick Start (5 minutes)

### Step 1: Setup (First Time Only)
```powershell
# Run interactive setup
.\Setup-Jules.ps1

# Follow the prompts:
# 1. Visit jules.google.com and create account
# 2. Connect your APEX repo
# 3. Generate API key
# 4. Paste key into script
# 5. Test connection
```

### Step 2: Run Analysis
```powershell
# Analyze your codebase
.\Analyze-Features-Jules.ps1

# Or with options:
.\Analyze-Features-Jules.ps1 -AutoApprove        # Skip manual approval
.\Analyze-Features-Jules.ps1 -SkipCommit         # Don't commit results
```

### Step 3: Review Results
1. Check the Jules session: `https://jules.google.com/u/1/session/{ID}`
2. Read the generated file: `conductor/feature_opportunities.md`
3. Review GitHub issue with summary
4. Discuss and prioritize with team
5. Create individual issues for approved features

## 📋 What Jules Does

When you run the analysis, Jules:

```
Your APEX Repository
        ↓
  [Jules AI Agent]
        ↓
  ✅ Scans all modules (CLI, RAM, MicroMax, IOT, etc.)
  ✅ Analyzes codebase structure and capabilities
  ✅ Identifies improvement opportunities
  ✅ Generates feature recommendations
  ✅ Assesses complexity and impact
  ✅ Suggests implementation priorities
        ↓
  Generated Artifacts:
  - conductor/feature_opportunities.md
  - GitHub Issue with summary
  - Jules session for interactive follow-up
```

### Example 4: Integration with GitHub Actions
```yaml
name: Weekly Jules Analysis

on:
  schedule:
    - cron: '0 2 * * 0'  # Every Sunday at 2 AM

jobs:
  analyze:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run Jules Analysis
        env:
          JULES_API_KEY: ${{ secrets.JULES_API_KEY }}
        shell: pwsh
        run: .\Analyze-Features-Jules.ps1 -AutoApprove
      - name: Push results
        run: |
          git config user.email "action@github.com"
          git config user.name "GitHub Action"
          git push origin master
```

## 🔐 Security Notes

### API Key Safety
- ✅ Stored in environment variables (not in code)
- ✅ Never committed to git
- ✅ Encrypted in Windows Credential Manager (if persistent)
- ✅ Can be rotated in Jules settings

### Best Practices
1. **Don't share your API key** with anyone
2. **Use environment variables** for storage
3. **Rotate keys** every 90 days
4. **Monitor usage** in Jules dashboard
5. **Review** all automated changes before merging

## 🆘 Troubleshooting

### "API key not found"
```powershell
# Set it:
$env:JULES_API_KEY = "your-key-here"

# Or make it permanent:
[Environment]::SetEnvironmentVariable("JULES_API_KEY", "your-key-here", "User")
```

### "Repository not found in Jules"
1. Go to https://jules.google.com
2. Install Jules GitHub app for APEX
3. Grant permissions
4. Try script again

### "Session timeout"
```powershell
# Increase timeout to 60 minutes:
.\Analyze-Features-Jules.ps1 -MaxWaitMinutes 60

# Or check manually:
https://jules.google.com/u/1/session/{SESSION_ID}
```

### "Git commit failed"
```powershell
# Check status:
git status

# Resolve conflicts, then retry
.\Analyze-Features-Jules.ps1 -SkipCommit

# Review, then commit manually:
git add conductor/feature_opportunities.md
git commit -m "Feature analysis update"
git push
```

See **JULES_GUIDE.md** for comprehensive troubleshooting.
