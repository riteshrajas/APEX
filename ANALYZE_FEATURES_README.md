# APEX Feature Analysis Tool

This PowerShell script analyzes the entire APEX repository (including submodules) and generates a comprehensive feature opportunities roadmap.

## What It Does

1. **Scans the Repository** — Analyzes Core/CLI, Core/RAM, MicroMax, IOT, and conductor modules
2. **Generates Feature Ideas** — Creates a comprehensive feature opportunities document
3. **Creates GitHub Issue** — Automatically creates a feature roadmap issue on GitHub
4. **Commits Markdown** — Commits the feature document to your repository

## Usage

### Basic Usage
```powershell
.\Analyze-Features.ps1
```

This will:
- Generate `conductor/feature_opportunities.md`
- Create a GitHub issue (requires `gh` CLI to be installed)
- Commit the file to git
- Push to GitHub (you'll be prompted if needed)

### With Options
```powershell
# Skip git commit
.\Analyze-Features.ps1 -NoCommit

# Skip GitHub issue creation
.\Analyze-Features.ps1 -NoIssue

# Skip both
.\Analyze-Features.ps1 -NoCommit -NoIssue
```

## Requirements

- **PowerShell** 5.0+ (Windows PowerShell or PowerShell Core)
- **Git** installed and configured
- **GitHub CLI** (`gh`) installed (optional, for auto-issue creation)
  - Install from: https://cli.github.com/

## Feature Categories

The script organizes feature ideas across:

1. **Core/CLI** — Voice features, tool extension system, memory management, debugging
2. **Core/RAM Dashboard** — Collaboration, analytics, workflow automation, knowledge base
3. **MicroMax/OS** — Sensor integration, hardware control, firmware updates, diagnostics
4. **MicroMax/OS Client** — UI improvements, device management, advanced control
5. **IOT Hardware** — PCB variants, thermal management, documentation
6. **System Integration** — Webhooks, data export, security enhancements
7. **Developer Experience** — Documentation, testing, tooling
8. **Product & Operations** — User management, deployment, community

## Customization

To modify the script:
1. Edit the `$featureContent` variable to adjust categories or features
2. Edit the `$issueBody` to change the GitHub issue template
3. Adjust commit messages as needed

## Tips

- Run this script **quarterly** to refresh the feature roadmap
- Use the generated issue for community discussion and prioritization
- Create individual GitHub issues for approved features
- Track implementation progress in project boards
- Gather community feedback before and after generation

## Example Workflow

```powershell
# 1. Generate fresh feature roadmap
.\Analyze-Features.ps1

# 4. Share the issue link with your community
# 5. Collect feedback and prioritize

```

## Troubleshooting

**"GitHub CLI not found"**
- Install from: https://cli.github.com/
- Or use `-NoIssue` flag to skip issue creation

**"Git not found"**
- Install Git from: https://git-scm.com/
- Or use `-NoCommit` flag to skip git operations

**"Merge conflicts"**
- The script handles merges automatically
- If issues occur, manually resolve in git and re-run
