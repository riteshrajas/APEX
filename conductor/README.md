# conductor

`conductor` is the project planning and execution-governance layer for APEX.

It centralizes product intent, implementation tracks, and workflow protocols used to drive consistent delivery.

## Navigation

- Root: [`../README.md`](../README.md)
- Context index: [`./index.md`](./index.md)
- Active tracks registry: [`./tracks.md`](./tracks.md)
- Workflow protocol: [`./workflow.md`](./workflow.md)

## Key Files

| File | Purpose |
| --- | --- |
| `index.md` | Entry point for project context |
| `product.md` | Product definition |
| `product-guidelines.md` | UX and product guardrails |
| `tech-stack.md` | Approved implementation stack |
| `workflow.md` | Task/phase lifecycle and quality gates |
| `tracks.md` | Registry of active tracks |
| `tracks/` | Per-track plans and execution artifacts |

## Usage Pattern

1. Start from `index.md`.
2. Select active work in `tracks.md`.
3. Open the specific track in `tracks/`.
4. Follow `workflow.md` while implementing.
